### Overview

The Patient Administration System (PAS) is an administrative system of record for hospital workflow. Non-clinical: identity, ADT, scheduling, waitlists, resources, communications, billing.

- ADT: admit → transfer → discharge with atomic bed allocation under `SELECT … FOR UPDATE`
- Scheduling: slot booking, cancellation, check-in, completion, no-show, bulk slot generation. **Recurring appointment series** (v0.9): RFC 5545 subset (daily/weekly/monthly + INTERVAL + BYDAY + COUNT/UNTIL); atomic per-patient-overlap reject; cap 200 occurrences.
- Waitlist + RTT clock with per-pathway breach threshold and `GET /api/waitlist/breaches`
- Resources: ward/bed occupancy snapshots, bed status state machine
- Billing: accounts, charges, invoices, payments — `rust_decimal` + ISO 4217. **Coverage** (v0.10): insurance / self-pay / other payer rows per patient with optional account linkage; FHIR R5 Coverage read at `/fhir/Coverage/{id}` and (v0.13) write via `POST /fhir` Bundle entries — same `CoverageRepository::create` path as the REST `POST /api/coverages`.
- Communication: Tera template letters with strict required-variable enforcement. SMS auto-send via the `SmsProvider` trait (v0.8) — `LogSmsProvider` ships for dev/test; consumer-supplied providers handle real gateways.
- Privacy: masked patient view, GDPR export, consent CRUD + revoke
- Audit log written transactionally with every state-changing call
- Transactional outbox + background dispatcher (2 s poll, batch 64). v0.5 bounds retries via `PAS_OUTBOX_MAX_RETRIES`: rows that exceed the budget move to `outbox_dead_letters` and can be replayed by an operator. v0.14 adds an HTTP webhook publisher (`PAS_WEBHOOK_URL` + optional HMAC-SHA256 via `PAS_WEBHOOK_SECRET`) and a `CompositePublisher` fan-out so HL7 v2 MLLP and webhook subscribers can both be active.
- Tantivy full-text patient search kept in sync with CRUD
- REST API (Axum) + FHIR R5 read/write for Patient / Encounter / Appointment / Practitioner / Schedule / Slot (last three v0.21) + read for Location
- FHIR R5 batch + transaction `POST /fhir` Bundles for Patient / Encounter / Appointment / Coverage (v0.13) / Practitioner / Schedule / Slot (v0.23) — transaction is genuinely all-or-nothing (every entry in one DB tx, search-index updates deferred to after commit)
- Bulk patient interchange in JSON / XML / TSV / CSV via the flat `PatientRow` projection
- HL7 v2 ADT — A01 / A02 / A03 / A04 / A05 / A06 / A07 / A08 / A11 / A12 / A13 / A21 / A22 / A23 / A28 / A38 / A40 — over both HTTP and MLLP TCP; exact-MRN dedup on inbound A01/A28; **A04 register outpatient** (v0.28) opens an Outpatient or Emergency encounter in Arrived status with no bed allocation, class derived from PV1-2; **A05 pre-admit** (v0.32) reserves the bed from PV1-3.3 and opens a Planned inpatient encounter (no admission row yet); **A38 cancel pre-admit** (v0.34) releases the reservation (bed → Available, encounter → Cancelled); **A21 / A22 leave of absence** (v0.36) brackets a temporary LOA — encounter `InProgress → OnLeave → InProgress`, bed stays Occupied; **A23 delete patient** (v0.39) soft-deletes a patient record (sent in error) — refused if patient has any open admission; **A06 change outpatient to inpatient** (v0.41) promotes the patient's active Outpatient/Emergency encounter to Inpatient with bed allocation; **A07 change inpatient to outpatient** (v0.43) demotes the open inpatient admission — releases bed, reclassifies encounter, status stays InProgress; **A12 cancel transfer** (v0.30) reverses the most-recent bed transfer, restoring the patient to the origin bed (same safety constraints as A13 cancel-discharge); batch envelope (`FHS`/`BHS`/`BTS`/`FTS`, v0.6) carries up to 1000 messages per transmission; A40 (v0.18) merges source MRN into survivor PID and is paired with the v0.11 `replaced_by` tombstone surface; outbound `Hl7v2MllpPublisher` when `HL7V2_OUTBOUND_PEER` is set (source-gated so REST-driven edits don't echo). **HL7 v2 DFT^P03** (v0.19 + v0.20) ingests one or more charges from FT1 segments against the patient identified by PID and auto-provisions an open billing account in the FT1-11.2 currency when none exists; v0.20 makes a multi-FT1 message all-or-nothing in one DB transaction. **HL7 v2 SIU — S12 (book) / S13 (reschedule) / S14 (modify) / S15 (cancel)** (v0.16 + v0.17) over the same HTTP + MLLP surface; inbound endpoints at `/api/hl7/v2/schedule-book` · `-reschedule` · `-modify` · `-cancel`; outbound mapping `AppointmentBooked → SIU^S12`, `AppointmentRescheduled → SIU^S13`, `AppointmentModified → SIU^S14`, `AppointmentCancelled → SIU^S15` with boomerang protection via `source: "hl7v2_*"` tags. **HL7 v2 MFN — M02 (Staff) / M05 (Patient Location)** (v0.24–v0.27) ingests MFI+(MFE+STF){1..N} or MFI+(MFE+LOC){1..N} for atomic master-file MAD/MUP/MDL on the practitioner or bed roster; outbound mapping `PractitionerCreated/Updated/Deactivated → MFN^M02` (v0.25) and `BedCreated/Updated/Retired → MFN^M05` (v0.27), both source-gated on `hl7v2_mfn_*`.
- Complete OpenAPI 3.1 spec for every REST + FHIR + HL7 v2 endpoint at `/api-docs/openapi.json` + Swagger UI at `/swagger-ui` (92 handlers across 19 tags)
- Ops dashboard at `GET /dashboard` (Tera + HTMX live-refresh + Lily Design System markup)
- Bearer-token auth (opt-in via `API_TOKEN`), configurable CORS, compression, request tracing
- Per-IP token-bucket rate limit (v0.12): default 600 req/min with burst 60, peer IP read from `ConnectInfo` (X-Forwarded-For not trusted), `/api/health` exempt, 429 + `Retry-After` + `RATE_LIMITED` envelope when exhausted; disable via `PAS_RATE_LIMIT_RPM=0`
- PostgreSQL via SeaORM; nested `pas-migrate` CLI; sibling read-mostly Loco-rs front-end at `../patient-administration-system-frontend` (port 5150)
- 497 unit tests + 62 integration tests across 24 files (most gated on `DATABASE_URL`)

Probabilistic patient matching, duplicate detection, and record merging *decisions* live in the sister MPI crate. The merge *application* (writing the tombstone, dropping search, FHIR link) lives here as of v0.11 — call `POST /api/patients/{id}/merge-into/{target_id}` once the MPI has picked a survivor.
